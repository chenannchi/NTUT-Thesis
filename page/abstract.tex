        \noindent \text 關鍵詞：大型語言模型代理人、自我修正機制、案例式學習、動態計畫修復

    \end{ZhAbstractItems}

    \begin{ZhAbstractDescription}
        大型語言模型代理人在執行 API 工具使用任務時，常因規劃不完整、參數缺失、認證錯誤、API 選擇不當或執行結果誤判而導致任務失敗。既有架構在遇到錯誤時多仰賴整體重新規劃，不僅容易浪費已成功執行的步驟，且可能重複產生相同錯誤。

        為改善此問題，本研究提出一套 API 任務代理人自我修正機制，結合案例式學習與動態計畫修復。系統透過結構化錯誤診斷產生可操作的修復回饋，並以 Rerollout 機制在失敗位置進行局部計畫調整，包括插入、替換或刪除修復步驟。此外，系統會將成功修復案例萃取為原子化經驗，並透過語意檢索於規劃、重新規劃與局部修復階段重複利用。

        實作上，本研究亦加入結構化計畫表示、依賴關係修正、Token 同步與可觀測化指標紀錄，以支援後續實驗分析。初步結果顯示，本機制能提供較細粒度的錯誤處理流程，並為評估代理人錯誤恢復能力與經驗復用效果建立可量化基礎。    \end{ZhAbstractDescription}
    
\end{ZhAbstract}
